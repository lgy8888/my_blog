# Git基础

# Git简介

  • Git是一个免费开源的分布式版本控制系统

  • 使用**仓库（Repository）**的数据库来记录文件的变化

  • 仓库中的每个文件中都有一个完整的版本历史记录

  • 可以看到谁在什么是修改了什么内容，还可以恢复到之前的版本

  • 目前流行的版本控制系统有两种
  • 集中式版本控制系统
    • 每次需要修改的时候就从中央服务器下载最新的版本
    • 修改后再将更新后的版本上传回中央服务器
    • 单点故障问题：中央服务器出现问题，全都无法修改

  • 分布式版本控制系统

    • 每个人的电脑上都有一个完整的版本库

    • 所以我们可以在本地上进行修改

    • 不存在单点故障问题

    • 需要分享的时候，只需要将仓库互相同步一下就可以了

    • 免费开源，速度快，功能强大，支持离线工作以及强大的分支管理

# 仓库Repo

  • 仓库中所有文件的修改，删除，添加等操作Git都能跟踪到

  • 本地文件生成Repo

    • 当前文件夹生成Repo
    git init
    • 当前文件夹下新建Repo
    git init my-repo

  • 从远程仓库clone

git clone https://github.com/geekhall-laoyang/remote-repo.git

# 工作区域和文件状态

# 三个工作区域

  • Git的本地数据管理分为三个区域
    • 工作区(Working Directory)
      • 本地目录
      • 本机上的文件夹
      • .git所在的目录
      • 车间
    • 暂存区(Staging Area/Index)
      • 临时存储区域
      • 用于保存即将提交到Git仓库的修改内容
      • .git/index
      • 货车
    • 本地仓库(Local Repository)
      • 通过git init创建的本地仓库
      • 包含完整的项目历史和元数据
      • Git存储代码和版本信息的主要位置
      • .git/objects
      • 仓库

# 四种文件状态

  • 未跟踪(Untrack)
    • 新创建的还没有被Git管理起来的文件
  • 未修改(Unmodified)
    • 已经被Git管理起来但是文件内容没有发生变化
  • 已修改(Modified)
    • 已经修改了文件但是没有添加到暂存区中
  • 已暂存(Staged)
    • 已经修改并已经添加到暂存区中
