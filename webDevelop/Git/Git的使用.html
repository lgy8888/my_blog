                                # 添加和提交

                                • 查看仓库的状态
                                  • 新增的未add的文件显示为红色 -> untracked files
                                git status

                                • 将文件添加到暂存区

                                  • add之后的文件显示为绿色
                                  git add

                                  • 批量添加文件到暂存区

                                    • 使用*通配符，将txt结尾的文件全都添加到暂存区
                                    git add *.txt
                                    • 使用.，将文件夹中所有文件全都添加到暂存区
                                    git add .

                                • 提交（将文件存入仓库）

                                  • 只提交暂存区中的内容，不会提交工作区中的内容
                                  • -m参数指定提交的信息
                                  • 若不使用-m指定提交的信息，则会进入vim编辑界面编辑要提交的信息
                                    • 方向键控制光标
                                    • 按i开启编辑
                                    • esc结束编辑
                                    • :wq保存
                                git commit -m "提交的信息"
                                • 在commit命令后面加上-a参数，表示添加到暂存区和提交一起进行
                                  • 可以省略为-am
                                git commit -a -m "提交的信息"
git commit -am "提交的信息"
                                • 查看仓库提交历史记录
                                  • 可以使用--oneline参数来查看简洁的提交记录
                                  • commit：十六进制唯一标识符
                                    • HEAD：指向当前版本
                                  • Author：提交的作者name <email>
                                  • Date：提交的时间
                                git log

                                git log --oneline
                                • 查看暂存区中的内容
                                git ls-files

                                # 回退版本

                                git reset V1
                                • 可以回退到之前的某个版本（提交的状态）
                                • git reset有三种不同的用法，分别对应三种不同的参数
                                  • --soft：表示回退到某一个版本，并保留工作区和暂存区的所有内容
                                  • --hard：表示回退到某一个版本，并丢弃工作区和暂存区的所有内容
                                  • --mixed：默认参数，表示回退到某一个版本，并保留工作区的内容，丢弃暂存区的内容
                                git reset --soft V2
                                • 回退完成 -> HEAD指向第二个提交记录
                                • ls查看工作区中文件 -> 第三次提交的文件依然存在
                                • git ls-files查看暂存区中的文件 -> 第三次提交的文件依然存在
                                • git status查看版本的状态 -> 第三次提交的文件显示为一个新文件
                                  • 对于回退完成的当前版本来说，工作区和暂存区中都有的第三次才提交的文件3.txt属于新文件
                                git reset --hard HEAD^
                                • 回退完成 -> HEAD指向第二个提交记录(HEAD^表示回退到上一个版本)
                                • ls查看工作区中文件 -> 第三次提交的文件不存在
                                • git ls-files查看暂存区中的文件 -> 第三次提交的文件不存在
                                • git status查看版本的状态 -> 没有文件
                                  • 对于回退完成的当前版本来说，工作区和暂存区中都不存在3.txt，版本状态中自然也不存在
                                git reset HEAD^

                                • 回退完成 -> HEAD指向第二个提交记录(HEAD^表示回退到上一个版本)

                                • ls查看工作区中文件 -> 第三次提交的文件依然存在

                                • git ls-files查看暂存区中的文件 -> 第三次提交的文件不存在

                                • git status查看版本的状态 -> 显示为红色的Untracked files

                                  • 对于回退完成的当前版本来说，工作区存在而暂存区不存在，说明该文件未被git管理上，这时git会将其识别为未跟踪文件

                                • 使用场景

                                  • 合并版本（soft，mixed）
                                  • 彻底删除版本（hard)

                                • 回溯操作

                                  • Git的所有操作都是可以回溯的
                                  • 查看操作历史记录
                                  git reflog
                                  • 回溯到某个版本
                                  git reset --hard V

                                # 查看差异

                                • 默认查看工作区与暂存区之间的差异
                                git diff
                                • 加上HEAD查看工作区和暂存区与仓库之间的差异
                                git diff HEAD
                                • 加上--cached参数或者--staged参数查看暂存区与仓库之间的差异
                                git diff --cached
git diff --staged
                                • 比较版本之间的差异
                                git diff <commit_hash1> <commit_hash2>   ---> 比较1和2之间的差异
git diff HEAD~ HEAD  					 ---> 比较上一版本和当前版本之间的差异
git diff HEAD^ HEAD  					 ---> 比较上一版本和当前版本之间的差异
git diff HEAD~2 HEAD 					 ---> 比较上上版本和当前版本之间的差异
                                • 比较分支之间的差异
                                git diff <branch_name1> <branch_name2>

                                # 删除文件

                                • 先从工作区删除文件，再暂存删除内容
                                rm file
git add file
                                • 把文件从工作区和暂存区同时删除
                                git rm <file>
                                • 把文件从暂存区删除，但保留在当前工作区中
                                git rm --cached <file>
                                • 递归删除某个目录下的所有子目录和文件
                                git rm -r *
                                • 删除后需要提交
                                git commit -m "delete xxx"

                                • .gitignore

                                # 应该忽略的文件

                                • 系统或者软件自动生成的文件
                                • 编译产生的中间文件和结果文件
                                • 运行时生成日志文件、缓存文件、临时文件
                                • 涉及身份、密码、口令、秘钥等敏感信息的文件

                                如何匹配

                                从上到下逐行匹配，每一行代表一个忽略模式

                                # 书写规则

                                • 空行或者以#开头的行会被Git忽略，一般空行用于可读性的分隔，#用于注释
                                • 使用标准的glob模式匹配，例如：
                                  • 星号*通配任意个字符
                                  • 问号?匹配单个字符
                                  • 中括号[]表示匹配列表中的单个字符
                                    • [abc]表示a或b或c
                                  • 两个星号**表示匹配任意的中间目录
                                  • 中括号可以使用短中线表示
                                    • [0-9]表示任意一位数字
                                    • [a-z]表示任意一位小写字母
                                  • 感叹号!表示取反

                                # 基本命令

                                • 查看分支列表
                                git branch
                                • 创建分支
                                git branch <分支名称>
                                • 切换分支（推荐使用switch）
                                git checkout <分支名称>
git switch <分支名称>
                                • 合并分支
                                git merge <分支名称>

                                • 删除分支

                                  • 已合并
                                  git branch -d <分支名称>
                                  • 未合并（强制删除）
                                  git branch -D <分支名称>

                                • 查看分支图

                                git log --oneline --graph --decorate --all
                                • 给命令起别名
                                alias graph="git log --oneline --graph --decorate --all"

                                # 变基和合并的区别

                                # Merge

                                • 优点：不会破坏原分支的提交历史，方便回溯和查看
                                • 缺点：会产生额外的提交节点，分支图比较复杂

                                # Rebase

                                • 优点：不会新增额外的提交记录，形成线性历史，比较直观和干净
                                • 缺点：会改变提交历史，改变了当前分支branch out的节点。
                                  • 避免在共享分支使用
