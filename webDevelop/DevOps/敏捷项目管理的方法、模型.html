                  # 敏捷项目管理的方法、模型

                  # 看板工具

                  看板工具的实质：后道工序在需要时，通过看板向前道工序发出信号，获取需要数量的输入，前道工序只有看到看板后，才按需生产。

                  看板信号由下游向上游传递，拉动上游的生产活动，使产品向下游流动。拉动源头是最下游的客户价值，也就是客户订单和需求

                  # Scrum

                  Scrum是一个轻量级的项目管理框架，它的核心在于迭代

                  # 三大特点

                  • 可能性的艺术（关注当下）
                  • 团队自组织，自管理（放权）
                  • 面对面的沟通

                  # Scrum框架

                  • ProductBacklog：产品待办列表
                  • SprintBacklog：迭代待办列表
                  • 在计划会议上，团队从产品列表当中，选择合适的条目进入到迭代待办列表
                  • 然后开始2-4周的迭代开发
                  • 迭代开发进行每天站立会议
                  • 迭代结束后提交一个潜在的可交付增量给用户评审
                  • 迭代最后团队会共同开展一次回顾会议，针对本次迭代的内容进行回顾
                  • ProductOwner：项目负责人
                  • ScrumMaster：在迭代过程中起到指导辅导的作用

                  # Scrum团队模型（三种角色）

                  • PO确定方向和目标
                  • ScrumMaster指导团队去实现目标

                  # Scrum三种工件

                  # Scrum过程模型

                  1. 从计划会议开始，PO需要准备好产品代办列表向团队进行介绍，确定迭代目标和合约，根据优先级和速率选择合适的条目进入Sprint待办列表 -> 产出迭代合约
                  2. 迭代开发的目标是可以工作的软件
                  3. 站立会议用作信息沟通，不解决问题不汇报
                  4. 迭代评审会议，邀请所有人参加，包括客户
                  5. 回顾会议，仅团队成员参与
