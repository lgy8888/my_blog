                                        # 华为敏捷项目管理企业实践

                                        # 敏捷迭代流程

                                        • Epic史诗，Feature特性，Story故事

                                        • leader将工作项目录到迭代计划中

                                        • 然后进入到迭代，迭代中按照scrum展开计划会议，每日站会，评审会议，回顾会议等活动

                                        # 项目管理服务

                                        • 典型的作业流场景，以及项目管理服务对应的功能分解

                                        # 新建项目

                                        • Scrum:
                                          • 标准的Scrum敏捷管理流程和实践
                                        • 看板
                                          • 轻量级，事务管理
                                        • DevOps全流程样例项目
                                          • 预置工作项，代码仓，一件可发布的在线商城样例项目

                                        # 需求规划

                                        • 思维导图的形式
                                          • 结构化分解需求
                                        • 图形化快速录入
                                        • 图形化快速拖动
                                          • 调整需求父子关系

                                        # 工作项管理

                                        • 填写工作项描述
                                        • 可指定模块，优先级，迭代，重要程度，抄送人等
                                        • 可指定预计工时

                                        # Backlog管理

                                        • 集中管理Story、缺陷和任务
                                        • 表格是交互
                                        • 快速创建子工作项

                                        # 迭代计划

                                        # 新建迭代计划

                                        • 管理所有迭代
                                        • 为迭代指定开始和结束时间
                                        • 为迭代命名

                                        # 工作项指定迭代

                                        • 可根据迭代导航选择查看当前迭代，未来迭代和过去迭代
                                        • 可查看不同迭代下的代交付工作项
                                        • 可拖动或者直接修改列表的迭代，为工作项指定迭代

                                        # 自定义工作流

                                        # 自定义工作项模板

                                        • 自定义添加字段
                                        • 可为字段指定不同的字段类型，灵活满足不同的场景

                                        # 自定义工作流

                                        • 自定义配置不同工作项的状态
                                        • 自定义调整状态的前后顺序

                                        # 自定义流转方向

                                        • 自定义状态的流转方向
                                        • 通过流转方向的控制，可以实现流程的走向控制

                                        # 自定义默认处理机制

                                        • 为特定状态自定义默认的处理人
                                        • 通过此设置，可实现指定状态默认走到特定的处理人，实现关键状态的审核集中处理
                                        • 通过此设置，可实现特定状态必须填写评论，如拒绝Bug必须填写拒绝原因等

                                        # 自定义通知

                                        • 自定义通知，可配置特定条件下通知到特定的人员
                                        • 支持站内信和邮件

                                        # 迭代回顾

                                        # Wiki在线协作，预制迭代回顾会议模板

                                        • Wiki轻量级在线协作，可用于需求分析，设计文档，会议纪要，迭代回顾等
                                        • 预制多种敏捷实践模板

                                        # 文档托管服务

                                        • 云端的项目文档托管
                                        • 支持同名文件的版本管理
                                        • 可在线预览多种文件格式
                                        • 支持和工作项管理，方便查看工作项的文档
                                        • 方便迭代回顾的总结和分享

                                        # 迭代回顾

                                        • 12种仪表盘Widget
                                        • 覆盖进度，质量，效率，成员工作项分布等，方便实时了解项目进展
                                        • 也用于迭代回顾，识别下一个迭代可以改进的方向
