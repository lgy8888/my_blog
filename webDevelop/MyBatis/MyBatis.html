# MyBatis

# MyBatis简介

  • MyBatis是一款持久层框架，用于简化JDBC开发
  • 持久层
    • 负责将数据保存到数据库的那一层代码
    • JavaEES三层架构：表现层，业务层，持久层
  • 框架：
    • 框架就是一个半成品软件，是一套可重用的，通用的，软件基础代码模型
    • 在框架的基础之上构建软件编写更加高效、规范、通用、可拓展

# 编写MyBatis核心配置文件

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
<!--
	typeAliases：类型别名，用于简化全限定名的书写
-->
    <typeAliases>
        <package name="com.LGYNB.pojo"/>
    </typeAliases>
<!--    
    environment：配置数据库连接环境信息，可以配置多个environment，通过default属性切换不同的environment
-->
    <environments default="development">
        <environment id="development">
            <transactionManager type="JDBC"/>
            <dataSource type="POOLED">
                <!--数据库连接信息-->
                <property name="driver" value="com.mysql.jdbc.Driver"/>
                <property name="url" value="jdbc:mysql:///db1?useSSL=false"/>
                <property name="username" value="root"/>
                <property name="password" value="123456"/>
            </dataSource>
        </environment>
    </environments>
    <mappers>
        <!--加载SQL映射文件-->
        <!-- <mapper resource="com/LGYNB/Mapper/XxxMapper.xml"/> -->
        <!--Mapper代理方式-->
        <package name="com.LGYNB.mapper"></package>
    </mappers>
</configuration>

# 编写SQL映射文件

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<!--
    namespace:名称空间
	id:类似方法名
	resultType:返回类型
	(namespace和resultType都需要使用全限定名)
-->
<mapper namespace="com.LGYNB.Mapper.XxxMapper">
    <!--
	无设置类型别名的写法
    <select id="selectAll" resultType="com.LGYNB.pojo.Xxx">
       	查询语句
    </select>
	-->
    <!--有设置类型别名的写法-->
    <select id="selectAll" resultType="com.LGYNB.pojo.Xxx">
        <!--查询语句-->
    </select>
</mapper>

# Mapper代理开发

  • 定义与SQL映射文件同名的Mapper接口，并且将Mapper接口和SQL映射文件放置在同一目录下

    • resource目录下的文件夹定义时需将.替换为/，才能达到分级建包

  • 设置SQL映射文件的namespace属性为Mapper接口全限定名

    • 全限定名：完整路径

      <mapper namespace="com.LGYNB.Mapper.XxxMapper"></mapper>

  • 在Mapper接口中定义方法，方法名就是SQL映射文件中SQL语句的id，并保持参数类型和返回值类型一致

  • 编码

    • 通过SqlSession的getMapper方法获取Mapper接口的代理对象

      XxxMapper mapper = sqlSession.getMapper(XxxMapper.class);

    • 调用对应方法完成SQL的执行

  • 若Mapper接口名称和SQL映射文件名称相同，并在同一目录下，则可以使用包扫描简化SQL映射文件的加载

# 参数占位符

  • #{}：执行SQL时，会将#{}占位符替换为？，将来自动设置参数值
  • ${}：拼SQL，会存在SQL注入问题
  • 使用：
    • 参数传递，都是用#{}
    • 如果要对表名，列名进行动态设置，只能进行${}进行SQL拼接

# SQL语句设置多个参数

  • 散装参数：需要使用@Param("SQL中参数占位符名称")
  • 实体类封装参数：只需要保证SQL中的参数名和实体类属性名对应上
  • map集合，只需要保证SQL中参数名和map集合的键的名称对应上

# MyBatis参数封装

  • 除了直接使用外，一律使用@Param注解来替换掉该map的键

  • 单个参数

    • pojo类型：直接使用，属性名 和 参数占位符名称一致

    • Map集合：直接使用，键名 和 参数占位符名称一致

    • Collection：封装为Map集合

      map.put("arg0", Collection);
map.put("collection", Collection);

    • List：封装为Map集合

      map.put("arg0", list);
map.put("collection", list);
map.put("list", list);

    • Array：封装为Map集合

      map.put("arg0", 数组);
map.put("array", 数组);

    • 其他类型：直接使用

  • 多个参数：封装为Map类型，@Param注解可以重命名该map的键

    map.put("arg0", 参数值1);
map.put("param1", 参数值2);
map.put("param2", 参数值3);
map.put("arg1", 参数值4);

# 注解完成增删改查

  • 使用注解完成简单功能，使用映射（配置文件）完成复杂功能
@Select("select * from tb_user where id = #{id}")
public User selectById(int id);
  • 查询：@Select
  • 插入：@Insert
  • 修改：@Update
  • 删除：@Delete

# SqlSessionFactory工具类

  • 声明静态变量
  • 利用静态代码块赋值
  • 声明静态方法返回工厂对象
public class SqlSessionFactoryUtils {
    private static SqlSessionFactory sqlSessionFactory;
    static {
        try {
            String resource = "mybatis-config.xml";
            InputStream inputStream = Resources.getResourceAsStream(resource);
            sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
        }catch (Exception e){
            e.printStackTrace();
        }
    }
    public static SqlSessionFactory getSqlSessionFactory(){
        return sqlSessionFactory;
    }
}
